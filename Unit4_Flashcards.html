Reuse Plan
Q: Define Reuse Plan. Explain Library, Framework, Pattern
  • Reuse Plan: Strategy to use existing components instead of building from scratch
Type Definition Example Control
Library Collection of pre-written functions/classes Math library, String library You call library
Framework Reusable architecture providing skeleton Spring, Django Framework calls you
Pattern Reusable solution template Singleton, Observer Design template
External Control Strategies
Q: Software Control Strategies & External Control Categories
Control Type Description Example
Interactive Interface User-driven through GUI (event-driven) Desktop applications, Web apps
Real-Time System Responds within strict time constraints Embedded systems, Control systems
Transaction Manager Manages database transactions (ACID) Banking systems, E-commerce
Dynamic Simulation Models real-world behavior over time Flight simulators, Training systems
Subsystems
Q: What is Subsystem? How to organize?
  • Subsystem: Package of classes working together for common purpose
  • Organization Methods:
  • 1. By Layers: Presentation ‚Üí Business Logic ‚Üí Data Access
  • 2. By Function: User Management, Payment, Reporting
  • 3. By Domain: Customer, Product, Order modules
  • Communication: Direct calls, Message passing, Shared data, Events
Data Storage Management
Q: Explain Data Storage Management
  • Deciding HOW and WHERE to store data persistently
  • Considerations:
  • 1. Data Format: Structured (tables) vs Unstructured (files)
  • 2. Access Patterns: Random vs Sequential
  • 3. Volume: Amount of data
  • 4. Performance: Read/Write speed
  • 5. Persistence: Temporary vs Permanent
  • 6. Security: Encryption, access control
Global Resource Handling
Q: How are Global Resources Handled?
  • Global Resources: Shared entities accessed by multiple parts
  • Handling Strategies:
  • 1. Singleton Pattern: Single instance accessible globally
  • 2. Resource Pooling: Reuse limited resources (connection pools)
  • 3. Synchronization: Control concurrent access (locks, semaphores)
  • 4. Resource Manager: Centralized allocation/deallocation control

Transformations
Q: Batch vs Continuous Transformation
Aspect Batch Transformation Continuous Transformation
Process Data processed in large groups at scheduled times Data processed as it arrives (real-time)
Timing Periodic (hourly, daily, monthly) Immediate, continuous
Advantages Efficient for large volumes, off-peak processing Real-time results, no waiting
Example Payroll processing, Database backups ATM transactions, Stock trading

System Performance & Application Class Model
Q: System Performance Estimation
  • Predicting system's speed, capacity, resource usage before implementation
  • Metrics: Response time, Throughput, Resource utilization
  • Methods: Analytical modeling, Simulation, Prototyping, Benchmarking
  • Helps identify bottlenecks early
Q: Constructing Application Class Model
  • Steps:
  • 1. Identify Classes: Find nouns in requirements
  • 2. Identify Attributes: Properties of each class
  • 3. Identify Methods: Actions/operations
  • 4. Identify Relationships: Associations between classes
  • 5. Define Multiplicity: How many instances
  • 6. Add Inheritance: Parent-child relationships
  • 7. Refine Model: Remove redundancy, add constraints
